# 0007: DRIFT遷移時の効果音

- **Status**: accepted
- **Date**: 2026-03-31
- **Deciders**: annenpolka
- **SPEC.md変更**: 「DRIFT突入時: 何もしない」→「DRIFT突入時: 短い効果音を再生」

## Context

SPEC.mdでは「通知は意思決定を強いるため出さない」として、DRIFT突入時に何もしない設計になっている。しかし実際に使うと、視覚的変化（翼のグローライン色変化）だけでは周辺視野で気づきにくい。聴覚フィードバックがないと、DRIFTに入ったことに気づかずAWAYまで落ちるケースが多い。

## Decision

FLOW→DRIFT遷移時に短い効果音を1回再生する。

### 音の要件

- **穏やかで短い**: 0.3〜0.5秒。判断を迫らない。水滴、木琴の一音、ソフトなベル
- **通知音ではない**: macOSの通知音やアラート音とは明確に異なる音色
- **音量**: システム音量の50%以下。存在に気づくが邪魔にならない
- **反復しない**: DRIFT突入時に1回のみ。DRIFT継続中の繰り返し再生はしない
- **無効化可能**: 設定でオフにできる

### SPEC.mdとの整合

SPEC.mdの「意志力を使わせない」原則は維持する。この効果音は:
- 行動を要求しない（「戻れ」ではなく「今ここ」の気づき）
- ポップアップやバナーを伴わない
- 意思決定を強いない（応答不要）

通知（notification）ではなく環境音（ambient cue）として位置づける。

## Options Considered

### Option A: システムサウンド使用（採用）

NSSound.play()でシステム内蔵音またはカスタム音声ファイルを再生。

- **Pro**: 実装が最もシンプル（1行）
- **Pro**: カスタム音声ファイルをバンドルすれば音色の完全制御が可能
- **Con**: システム音量に依存

### Option B: AVAudioEngineでトーン生成

正弦波をプログラマティックに生成して再生。

- **Pro**: 外部ファイル不要
- **Pro**: 周波数・エンベロープを精密制御
- **Con**: 実装が複雑
- **Con**: AVAudioEngine初期化のオーバーヘッド

### Option C: 効果音なし（SPEC.md準拠）

視覚的変化のみ。

- **Pro**: SPEC.md通り
- **Con**: 気づきにくい

## Consequences

- SPEC.md「DRIFT突入時: 何もしない」の注記に「ADR-0007により効果音を追加」を記載
- SitboneCore/SessionEngineに音声再生のコールバックまたはデリゲートを追加
- 設定画面に「DRIFT効果音: ON/OFF」を追加（v0.2）
- AWAY遷移時の効果音は**追加しない**（SPEC.md「音もポップアップもなし」を維持）
