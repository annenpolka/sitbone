# 0020: ノッチからのセッション開始・停止

- **Status**: accepted
- **Date**: 2026-04-17
- **Deciders**: annenpolka

## Context

現状、セッション開始/終了のエントリポイントはMenuBarExtraの Start Session / End Session ボタン（⌘⇧S）のみ。ノッチオーバーレイ自体は非セッション時も描画されているが、ウィング・ドロップダウンの中身は `if engine.isSessionActive` で条件分岐され、非アクティブ時は空に塗り潰される（ADR-0006 の「非セッション時: notchのみ」方針）。

これは次の問題を生んでいる:

1. ノッチは常に視界の中央上部にあり、ユーザーはそこに視線を向ける習慣が既にある。開始するためにわざわざメニューバーアイコンをクリックする必要があるのは、周辺視野で完結する設計思想と整合しない。
2. `auto-start` オプションを外した後や、`endSession()` した直後の再開ハードルが高い。
3. プロファイル切替はノッチドロップダウンで完結するのに、その隣で「今から何を計測するか」を決めるボタンだけがノッチに存在しない。

ADR-0006 は「非セッション時: オーバーレイ非表示、CPUゼロ」と書いているが、実装では非アクティブ時も描画は続いており（ウィングの黒い翼形と空のドロップダウンエリアは存在する）、CPUコストは描画ではなくホバーポーリング（30Hz）が支配している。つまり「何も表示しない」よりも「開始動線を置く」方が既存コストの範囲内で実現できる。

## Decision

ノッチドロップダウン内にセッション開始・停止の両ボタンを置く。

- **非セッション時**: プロファイルPills + 大きな「▶ Start Session」キャップセルボタン。クリックで `engine.startSession()`
- **セッション中**: プロファイルPills の右端に小さな「■」停止アイコン。クリックで `engine.endSession()`
- 開始ボタンは「これから始める決意」を視覚的に強調するためプロファイルカラーで塗る。停止ボタンは現在進行中の統計を奪わないよう、プラスボタン同等のサイズにとどめる（視覚的重みを揃える）
- プロファイルPillsはどちらのモードでも表示（開始前の選択、途中でのプロファイル切替、どちらもノッチ内で完結する）
- キーボードショートカットは MenuBar の `⌘⇧S` をそのまま維持（ノッチボタンはマウス操作の補助として位置付ける）
- ADR-0006 の「非セッション時はノッチのみ」の記述を「ドロップダウンは両モードで開始/停止の導線を持つ」に更新する（本ADRで上書き）

## Options Considered

### Option A: ドロップダウン内に Start / End ボタンを追加（採用）

ホバーで現れるドロップダウン内に、モード別の操作を配置する。Start はセッション未開始時の大ボタン、End はセッション中の小アイコンボタン。

- **Pro**: 既存のホバー導線（30Hz ポーリング）に乗るので、新しい検知系を追加しなくてよい
- **Pro**: プロファイルPillsと並んで表示されるので、「どのプロファイルで始めるか」「今停止するか」を同じ画面で決められる
- **Pro**: マウス操作を奪わない（`nonactivatingPanel` のまま、ADR-0004 の方針維持）
- **Pro**: Start と End の視覚的重みを変えることで、開始は意図的な行為、停止は軽いリセット操作として区別できる
- **Con**: ホバーしないと見えない。初見のユーザーが開始・停止方法に気づかない可能性がある（→ MenuBar に残す Start/End Session と `⌘⇧S` で補完）
- **Con**: End を小さなアイコンにしたので誤タップは少ないが、密集時にプロファイル追加の「＋」ボタンと視覚的に近くなる（サイズと色で区別）

### Option B: ウィング自体をクリックで開始

左右のウィングを非セッション時だけ `ignoresMouseEvents=false` にしてタップで開始。

- **Pro**: 一段階操作で済む（ホバー→クリック ではなく 直接クリック）
- **Con**: ウィングは 14pt 幅 × ノッチ高さしかなくクリック領域が小さい
- **Con**: ウィングはセッション中に状態バーを表示するので、モードによって意味が変わる設計は分かりにくい
- **Con**: `interactive: false` 前提で作られた `leftPanel`/`rightPanel` の再設計が必要

### Option C: 常時クリックで開始可能なコンパクトボタンをノッチ下に固定表示

ホバー非依存で、小さな play アイコンを常時表示。

- **Pro**: 発見しやすい
- **Con**: Dynamic Island ライクな「何もないときは何もない」設計（ADR-0006 Option A）を壊す
- **Con**: アイコンが常時存在すると、集中中の視線を奪う副作用がある

## Consequences

- `NotchDropdown.body` の `if engine.isSessionActive` ブロックに `else` が追加される。非アクティブ時も `contentHeight` が preference で伝搬するため、ホバー継続領域が確保される（クリックを取りこぼさない）
- `ProfilePills` にジェネリックな `trailing` スロットを設け、セッション中は `EndSessionButton` を差し込み、非セッション時は `EmptyView` を差し込む。profile pills のレイアウトは両モードで共有される
- `engine.startSession()` / `engine.endSession()` のエントリポイントがそれぞれ2箇所になる。テストは既存の `SessionEngine` 契約を信頼する（UIレイアウトはSnapshot範囲、ロジック重複はない）
- MenuBar の `sessionToggle` は残す。`⌘⇧S` 含め操作性は変わらない
- ADR-0006 の「非セッション時: オーバーレイ非表示、CPUゼロ」は厳密には嘘だったが、本ADRで「非セッション時: 開始ボタン / セッション中: 停止アイコンと統計」に改める
- 誤って End ボタンを押した場合の即時 Undo は本ADR範囲外（将来のADRで扱う）
